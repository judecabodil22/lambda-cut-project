# Lambda Cut

Automated pipeline to convert long-form YouTube streams into shorts with AI-generated scripts and TTS narration.

## How It Works

```
YouTube Playlist → Download → Transcribe → AI Scripts → Video Clips → TTS Audio + Subtitles
     Phase 1         Phase 2      Phase 3      Phase 4       Phase 5
```

Each phase can be run independently or skipped. The pipeline supports checkpointing — re-running picks up where it left off.

## First Time Setup

Run the setup wizard. It checks dependencies, validates your credentials, and tests all connections before you start:

```bash
./workflows/onboard.sh
```

The wizard will:
1. Check system dependencies (python3, ffmpeg, yt-dlp, stable-ts, curl)
2. Verify Chrome has YouTube cookies (needed for downloads)
3. Prompt for your API keys, bot token, chat ID, and playlist URL
4. Validate each input format before accepting it
5. Test the Gemini API, Telegram bot, and YouTube playlist access
6. Write `.env` and `gemini_keys.txt` with proper permissions

You can re-run it anytime to update config — existing values are shown as defaults.

### What You Need

| Item | Where to Get It |
|------|-----------------|

## Running the Pipeline

```bash
# Full pipeline (all 5 phases)
./workflows/lambda_cut.sh

# Specific phases only
./workflows/lambda_cut.sh -phase 2          # transcribe only
./workflows/lambda_cut.sh -phase 2,3        # transcribe + scripts
./workflows/lambda_cut.sh -phase 5          # TTS only

# Skip phases (use existing outputs)
./workflows/lambda_cut.sh -skip-phase-1 -skip-phase-2
./workflows/lambda_cut.sh -skip-all         # dry run with existing files
```

## Pipeline Phases

### Phase 1 — Video Download
Downloads the latest video from your playlist using `yt-dlp` with `--cookies-from-browser chrome`. Retries 3 times on failure.

**Output:** `streams/*.webm`

### Phase 2 — Transcription
Runs `stable-ts` with voice activity detection, forced English language, and garbage segment cleaning (removes repetitive tokens, single-word loops, noise).

**Output:** `transcripts/*.json`, `transcripts/*.srt`

### Phase 3 — Script Generation
Sends each hour's transcript to Gemini 2.5 Flash-Lite. Generates a first-person narrative script with direct quotes and a cliffhanger ending. Uses key rotation across multiple API keys with rate limiting.

**Output:** `scripts/script_001.txt`, `script_002.txt`, ...

### Phase 4 — Clip Generation
Finds the most dialogue-dense scenes per hour using `smart_extract.py`. Cuts clips with variable length (1-5 min) using VAAPI hardware encoding (AMD GPU) or CPU fallback. Applies loudnorm audio normalization.

**Output:** `shorts/short_001_1.mp4`, `short_001_2.mp4`, ...

### Phase 5 — TTS Generation
Sends each script to Gemini TTS (24kHz mono PCM), converts to WAV (44.1kHz stereo), then transcribes the WAV to SRT subtitles.

**Output:** `tts/tts_001.wav`, `tts/tts_001.srt`, ...

## Configuration

### Environment Variables

Set in `workflows/.env` (generated by `onboard.sh`):

| Variable | Default | Description |
|----------|---------|-------------|
| `GEMINI_API_KEY` | required | Gemini API key (TTS) |
| `TELEGRAM_BOT_TOKEN` | required | Telegram bot token |
| `TELEGRAM_CHAT_ID` | required | Telegram chat ID |
| `PLAYLIST_URL` | required | YouTube playlist URL |
| `TTS_VOICE` | Algenib | Voice name (see list below) |
| `TTS_STYLE` | (empty) | Style prefix for TTS |
| `SCRIPT_DELAY` | 300 | Seconds between script generations |
| `TTS_DELAY` | 300 | Seconds between TTS generations |

### API Key Rotation

Place multiple Gemini keys in `workflows/gemini_keys.txt` (one per line). The pipeline rotates keys across hours to avoid rate limits. Exponential backoff on 429 errors (15s, 30s, 60s). Falls back to raw transcript if all keys fail.

## Troubleshooting

| Issue | Cause | Fix |
|-------|-------|-----|
| Chrome cookies not working | Chrome is running | Close Chrome or use `--cookies-from-browser chrome --cookies-from-browser-mode none` |
| Gemini 429 rate limit | Quota exceeded | Add more keys to `gemini_keys.txt`, increase `SCRIPT_DELAY`/`TTS_DELAY` |
| Telegram not responding | Listener not running | Start `./workflows/telegram_listener.sh` in a separate terminal |
| Transcription empty | No speech in video | Check video content; adjust `--vad_threshold` in the script |
| Download fails | YouTube rate limit | Pipeline retries 3x; update `yt-dlp` |
| System sleeps during pipeline | Inhibition not active | Pipeline uses `systemd-inhibit`; re-run if interrupted |
